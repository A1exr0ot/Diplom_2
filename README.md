*   # Автоматизация API-тестов Stellar Burgers (Задание 2)

Данный проект содержит набор автоматизированных тестов для REST API сервиса космической бургерной **Stellar Burgers**. Тестирование покрывает основные бизнес-процессы: управление жизненным циклом пользователя и создание заказов.

## Используемый технологический стек
* **Java 11** - основной язык разработки.
* **REST Assured (4.4.0)** - библиотека для автоматизации тестирования REST-сервисов.
* **JUnit 4.13** - фреймворк для написания и запуска тестов.
* **Lombok 1.18.30** - библиотека для сокращения шаблонного кода (использована для моделей данных/POJO).
* **Allure (2.23)** - инструмент для генерации детализированных отчетов (использованы аннотации `@Step`, `@DisplayName`, `@Description`).
* **Maven** - система сборки и управления зависимостями.
* **Apache HttpCore** - использование констант `HttpStatus` вместо «магических чисел».

## Архитектура проекта
Проект построен по принципу разделения ответственности:
* **Models (POJO)**: Классы данных (`User`, `Order`, `Ingredients`), оптимизированные с помощью Lombok (`@Data`, `@AllArgsConstructor`).
* **Clients**: Логика HTTP-запросов вынесена в отдельные классы-клиенты, что обеспечивает чистоту тестовых сценариев.
* **Tests**: Сами тестовые классы, наследующиеся от `BaseTest` для централизованной инициализации клиентов.

## Покрытые сценарии
1. **Создание пользователя**:
    * Создание уникального пользователя.
    * Ошибка при создании дубликата.
    * Валидация обязательных полей (email, пароль, имя) с проверкой текста ошибки в теле ответа.
2. **Логин пользователя**:
    * Успешный вход под существующим пользователем.
    * Ошибка при входе с неверным логином/паролем.
3. **Создание заказа**:
    * Создание заказа с авторизацией и без.
    * Создание заказа с валидными ингредиентами.
    * Проверка ошибок при создании заказа без ингредиентов или с неверным хешем.

## Очистка данных
В проекте реализована автоматическая очистка данных: в методах `@After` выполняется удаление созданных тестовых пользователей, чтобы не засорять базу данных сервиса.
